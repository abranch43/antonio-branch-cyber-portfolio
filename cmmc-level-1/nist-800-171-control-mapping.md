# NIST SP 800-171 Control Mapping

## Purpose

This sample mapping shows how cybersecurity and GRC work can translate control families into implementation approaches, evidence types, and readiness status. It is an employer-facing artifact and does not include client data, passwords, confidential system details, or private contract records.

Antonio Branch is the Founder & CEO of A+ Enterprise LLC, a certified SDVOSB, MBE, and SDVE focused on audit-ready public operations, compliance documentation, CMMC readiness, NIST SP 800-171 control mapping, incident response documentation, vulnerability management reporting, and automation-enabled execution through BranchOS.

A+ Enterprise LLC maintains a CMMC Level 1 self-assessment in SPRS under UID S100053243 for FCI/basic safeguarding alignment under FAR 52.204-21. Level 2 readiness will be evaluated only where CUI or contract-specific requirements apply.

## Sample Control Mapping

| Control Family | Implementation Approach | Evidence Type | Status |
| --- | --- | --- | --- |
| Access Control | Limit access to authorized users, assign roles by business need, review user access periodically, and remove access during offboarding. | Access review checklist, account approval record, offboarding checklist, MFA settings summary | Implemented / monitored |
| Awareness and Training | Provide cybersecurity awareness guidance covering phishing, data handling, incident reporting, and acceptable use. | Training log, acknowledgment record, security awareness checklist | Implemented / refresh scheduled |
| Audit and Accountability | Track security-relevant activity where supported by the system and retain review notes for accountability. | Log review notes, audit settings summary, exception review record | Partially implemented / improving |
| Configuration Management | Maintain approved software, document configuration changes, and validate security-relevant settings after change. | Configuration baseline, change log, approved software list, validation checklist | Implemented / monitored |
| Identification and Authentication | Require unique user identities, strong authentication practices, and MFA where available. | User account list, MFA summary, password policy record, identity provider settings | Implemented / monitored |
| Incident Response | Maintain a playbook for reporting, triage, containment, recovery, evidence handling, and lessons learned. | Incident response playbook, severity matrix, incident intake form, post-incident review template | Implemented / exercised through tabletop |
| Media Protection | Control storage, handling, transfer, and disposal of media that may contain sensitive business or government information. | Media handling procedure, disposal checklist, storage location review | Drafted / needs periodic review |
| Risk Assessment | Identify risks, assess likelihood and impact, document remediation, and track risk acceptance decisions. | Risk register, vulnerability summary, remediation tracker, risk acceptance memo | Implemented / monitored |
| Security Assessment | Review control performance, document gaps, assign owners, and track corrective actions. | Self-assessment summary, plan of action, management review notes | Implemented / recurring review |
| System and Communications Protection | Use secure communications, protect remote access, and document network or system safeguards. | VPN or secure access summary, encryption settings, firewall rule review notes | Implemented where applicable |
| System and Information Integrity | Identify vulnerabilities, apply patches, review endpoint protections, and track remediation through closure. | Vulnerability scan report, patch log, endpoint protection summary, follow-up verification | Implemented / monitored |

## Notes for Reviewers

This table is a sample public artifact. It demonstrates control analysis and evidence planning, not a claim of CMMC Level 2 certification. Where CUI, export-controlled data, contract-specific security clauses, or higher assurance requirements apply, scope and readiness must be reviewed against the actual contract and system boundary.
