# Cyber Certification Plan

## Purpose

This certification roadmap aligns cybersecurity learning with practical Cybersecurity Analyst, GRC Analyst, SOC Analyst, IAM, and cloud security roles. It is designed to build credibility without overstating certifications that are still planned.

Antonio Branch is the Founder & CEO of A+ Enterprise LLC, a certified SDVOSB, MBE, and SDVE focused on audit-ready public operations, compliance documentation, CMMC readiness, NIST SP 800-171 control mapping, incident response documentation, vulnerability management reporting, and automation-enabled execution through BranchOS.

## Roadmap Summary

| Certification | Primary Role Alignment | Purpose |
| --- | --- | --- |
| ISC2 Certified in Cybersecurity | Entry-level Cybersecurity Analyst, GRC Analyst | Establish broad cybersecurity foundation and professional vocabulary. |
| CompTIA Security+ | Cybersecurity Analyst, GRC Analyst, SOC Analyst | Validate core security concepts, risk management, incident response, IAM, and operations knowledge. |
| CompTIA CySA+ | SOC Analyst, Vulnerability Management Analyst, Detection and Response | Build stronger analysis skills for alerts, vulnerabilities, threat behavior, and response workflows. |
| Microsoft SC-900 or AZ-500 | IAM, Cloud Security, Microsoft Security Operations | Connect security fundamentals or implementation skills to Microsoft cloud and identity environments. |
| Later CISSP Pathway | Senior GRC, Security Manager, Security Program Leadership | Long-term pathway after experience requirements are met and broader security leadership responsibilities mature. |

## Phase 1: ISC2 Certified in Cybersecurity

Role targets:

- Cybersecurity Analyst
- GRC Analyst
- Security compliance support
- Junior security operations support

Why it matters:

ISC2 Certified in Cybersecurity supports baseline understanding of security principles, business continuity, access control, network security, and incident response. It is a practical first credential for demonstrating commitment to the cybersecurity field.

Portfolio alignment:

- Incident response playbook
- Evidence binder overview
- Vulnerability management sample report
- Security awareness and access control documentation

## Phase 2: CompTIA Security+

Role targets:

- Cybersecurity Analyst
- GRC Analyst
- SOC Analyst
- Security operations coordinator

Why it matters:

Security+ is widely recognized for foundational security knowledge across threats, architecture, implementation, operations, governance, risk, and compliance. It supports employer screening for analyst roles and strengthens the language used in security documentation.

Portfolio alignment:

- NIST SP 800-171 control mapping
- Risk and remediation tracking
- Incident response severity model
- Vulnerability report remediation plan

## Phase 3: CompTIA CySA+

Role targets:

- SOC Analyst
- Vulnerability Management Analyst
- Cybersecurity Analyst
- Detection and response support

Why it matters:

CySA+ emphasizes analysis, detection, vulnerability management, incident response, and reporting. It is a natural next step after Security+ for roles that require reviewing alerts, scanning results, logs, and remediation evidence.

Portfolio alignment:

- Vulnerability scan report sample
- Follow-up verification model
- Incident detection and analysis workflow
- BranchOS vulnerability tracker concept

## Phase 4: Microsoft SC-900 or AZ-500

Role targets:

- IAM Analyst
- Cloud Security Analyst
- Microsoft security support
- GRC Analyst supporting Microsoft environments

Why it matters:

SC-900 provides Microsoft security, compliance, and identity fundamentals. AZ-500 is more technical and implementation-focused for Azure security. The best path depends on target roles:

- Choose SC-900 first for security, compliance, and identity fundamentals.
- Choose AZ-500 when ready to demonstrate hands-on Azure security implementation.

Portfolio alignment:

- Access control monitor concept
- MFA and identity review documentation
- Compliance dashboard module design
- Cloud security evidence planning

## Phase 5: Later CISSP Pathway

Role targets:

- Senior GRC Analyst
- Security Manager
- Security Program Lead
- Public-sector cybersecurity leadership

Why it matters:

CISSP is a long-term security leadership credential. It should be pursued when experience requirements and role responsibilities support it. Until then, it should be described as a future pathway, not a current credential.

Portfolio alignment:

- Compliance governance model
- Control family mapping
- Incident response lifecycle
- Risk management documentation
- Audit-ready operating system design

## Execution Plan

| Timeframe | Focus | Output |
| --- | --- | --- |
| 0-60 days | ISC2 Certified in Cybersecurity study and exam preparation | Foundational credential target and updated resume language after completion |
| 60-150 days | CompTIA Security+ preparation | Stronger analyst screening credential target |
| 150-270 days | CySA+ preparation and practical reporting samples | More mature SOC and vulnerability management alignment |
| 270-360 days | SC-900 or AZ-500 based on role target | Microsoft identity, compliance, or cloud security alignment |
| Later | CISSP pathway planning | Leadership credential roadmap after experience requirements are satisfied |

## Portfolio Discipline

This roadmap should be updated only when certifications are earned or exam dates are formally scheduled. Planned certifications should remain clearly marked as planned to avoid overstating qualifications.
